                                                    Parte didáctica

                                                    Definiciones

                                                    Las métricas de clasificación no sirven solo para decir si un modelo acierta mucho. Sirven para entender en qué se equivoca, qué tipo de error comete y si ese error importa.

                                                    Contexto histórico

                                                    Las métricas de clasificación no nacieron con el machine learning. Tienen raíces en la teoría de la detección de señales, desarrollada durante la Segunda Guerra Mundial para distinguir entre ruido y señales reales en radar y sonar.

                                                    En los años 50 y 60, estos conceptos migraron a la medicina diagnóstica, donde se necesitaba medir la fiabilidad de pruebas clínicas. Los médicos hablaban de sensibilidad, lo que hoy llamamos Recall, y especificidad, relacionada con la tasa de verdaderos negativos.

                                                    Con el auge del machine learning en los años 90 y 2000, estas métricas se estandarizaron como lenguaje común para evaluar modelos de clasificación.

                                                    Mensaje clave: esto no es una invención reciente. Es una tradición científica de décadas para medir la fiabilidad de sistemas que toman decisiones binarias.

                                                    Confusion Matrix

                                                    La Confusion Matrix, o matriz de confusión, es una tabla de cuatro casillas que permite visualizar el rendimiento de un algoritmo de clasificación supervisada. Cada fila representa la clase real y cada columna representa la clase predicha por el modelo.

                                                    Dicho de forma sencilla: cuenta cuántas veces el modelo acertó y se equivocó, y de qué tipo de error se trata.

                                                    True Positive · TP
                                                    Acierto: detecté un positivo real.
                                                    False Positive · FP
                                                    Error: dije “sí”, pero era “no”. Es una falsa alarma.
                                                    True Negative · TN
                                                    Acierto: descarté correctamente un negativo.
                                                    False Negative · FN
                                                    Error: dije “no”, pero era “sí”. No detecté lo importante.

                                                    Memotecnia: la primera palabra indica si el modelo acertó. True significa que sí; False significa que no. La segunda palabra indica qué predijo el modelo. Positive significa que dijo “positivo”; Negative significa que dijo “negativo”.

                                                    Accuracy --

                                                    La Accuracy es la proporción de instancias correctamente clasificadas sobre el total. Responde a la pregunta: de todas las decisiones, ¿cuántas acertó el modelo?

                                                    Fórmula: (TP + TN) / (TP + TN + FP + FN).

                                                    Cuando juegues una partida, aquí aparecerá la interpretación con vuestros datos.

                                                    Recall --

                                                    El Recall mide la proporción de positivos reales que fueron correctamente identificados. Responde a la pregunta: de todos los casos importantes que existían, ¿cuántos detecté?

                                                    Fórmula: TP / (TP + FN).

                                                    Importa especialmente cuando el coste de no detectar algo es alto: medicina, fraude, seguridad o riesgos críticos.

                                                    Cuando juegues una partida, aquí aparecerá cuántos asesinos reales consiguió descubrir el pueblo.

                                                    Precision --

                                                    La Precision mide la proporción de predicciones positivas que fueron correctas. Responde a la pregunta: cuando el modelo dice “sí”, ¿cuántas veces acierta?

                                                    Fórmula: TP / (TP + FP).

                                                    Importa cuando una falsa alarma tiene coste: bloquear una tarjeta legítima, tratar a una persona sana o acusar a un inocente.

                                                    Cuando juegues una partida, aquí aparecerá si el pueblo fue fiable al expulsar jugadores.

                                                    F1-Score --

                                                    El F1-Score es la media armónica entre Precision y Recall. Penaliza los valores extremos: si una de las dos métricas es muy baja, el F1 cae.

                                                    Fórmula: 2 × (Precision × Recall) / (Precision + Recall).

                                                    Sirve para resumir el equilibrio entre detectar lo importante y ser fiable cuando se acusa.

                                                    Cuando juegues una partida, aquí aparecerá el equilibrio real entre encontrar asesinos y no expulsar vecinos por error.

                                                    El dilema: Recall vs Precision

                                                    Normalmente no puedes maximizar ambas a la vez solo moviendo el umbral de decisión. Si el modelo es muy estricto, sube la Precision, pero baja el Recall porque deja escapar positivos reales. Si el modelo es muy permisivo, sube el Recall, pero baja la Precision porque genera más falsas alarmas.

                                                    Sí pueden subir ambas métricas si mejoras el modelo de verdad: mejores datos, mejor algoritmo, menos ruido o mejor representación del problema.

                                                    Valores óptimos y rangos de interpretación

                                                    Accuracy
                                                    Puede sonar bien por encima de 0,9, pero puede engañar si las clases están desbalanceadas.
                                                    Recall
                                                    1,0 significa que se detectaron todos los positivos reales. 0 significa que no se detectó ninguno.
                                                    Precision
                                                    1,0 significa que cada vez que el modelo dijo “positivo”, acertó. 0 significa que todas sus alertas fueron falsas.
                                                    F1-Score
                                                    1,0 exige Precision y Recall perfectos. Si una de las dos cae a 0, el F1 también cae a 0.

                                                    No existe un valor óptimo universal. La pregunta correcta no es “¿mi F1 es alto?”, sino: “¿este F1, con esta Precision y este Recall, es adecuado para el problema que estoy resolviendo?”.

                                                    ¿Cuándo usar cada métrica?

                                                    Clases equilibradas y errores parecidos
                                                    Accuracy puede ser suficiente.
                                                    No detectar algo es grave
                                                    Recall debe tener prioridad.
                                                    Las falsas alarmas son costosas
                                                    Precision debe tener prioridad.
                                                    Necesitas equilibrio
                                                    F1-Score ayuda a comparar modelos sin mirar solo una métrica.

                                                    Aplicación profesional

                                                    Casos reales

                                                    Las métricas no son números decorativos para un informe. Son decisiones que afectan a personas, recursos, costes y riesgos reales.

                                                    Caso 1: Detección de cáncer de mama con IA

                                                    En medicina, el coste de un falso negativo puede ser muy alto. Si un sistema no detecta un cáncer real, el paciente puede perder tiempo crítico para el diagnóstico y tratamiento.

                                                    Por eso, en muchos contextos médicos se prioriza el Recall: interesa detectar la mayor cantidad posible de casos positivos reales. Un falso positivo puede generar ansiedad o pruebas adicionales, pero un falso negativo puede ser mucho más peligroso.

                                                    La lección es que elegir la métrica correcta no es solo técnico. Es ético y práctico.

                                                    Caso 2: Detección de fraude bancario

                                                    Un banco puede usar un modelo de clasificación para detectar transacciones fraudulentas.

                                                    Si prioriza mucho el Recall, detectará casi todos los fraudes, pero bloqueará también muchas operaciones legítimas. Eso provoca llamadas al servicio de atención al cliente, frustración y pérdida de confianza.

                                                    Si prioriza mucho la Precision, solo bloqueará cuando esté muy seguro, pero se le pueden colar fraudes reales.

                                                    En la práctica, muchos sistemas usan bandas de riesgo: bloqueo automático para riesgo alto, verificación adicional para riesgo medio y paso directo para riesgo bajo.

                                                    Caso 3: Filtro de spam

                                                    Un filtro de spam decide si un mensaje debe llegar a la bandeja de entrada o quedarse en spam.

                                                    Si prioriza demasiado el Recall, casi no se cuela spam, pero también puede ocultar correos importantes: facturas, invitaciones, recordatorios médicos o mensajes de trabajo.

                                                    Si prioriza demasiado la Precision, marcará como spam solo lo muy evidente, pero la bandeja se llenará de publicidad.

                                                    Por eso muchos filtros combinan varios modelos, reglas y aprendizaje del comportamiento del usuario.

                                                    Mensaje central

                                                    Recall bajo en medicina puede significar vidas perdidas. Precision baja en banca puede significar clientes bloqueados injustamente. Un mal equilibrio en spam puede significar pérdida de productividad.

                                                    El científico de datos no solo elige un modelo. Elige qué tipo de error es aceptable y para quién.

                                                    Reflexión ética

                                                    Debate

                                                    Hemos visto que F1 resume el equilibrio de un modelo. Pero un F1 global puede esconder diferencias importantes entre regiones, grupos o poblaciones.

                                                    Pregunta elegida

                                                    Un modelo de predicción de riesgo de incendios forestales tiene F1 del 72% a nivel nacional. Al desglosar por regiones, en Galicia es del 85% y en Extremadura del 38%. ¿Es ético desplegarlo nacionalmente?

                                                    Por qué esta pregunta funciona

                                                    Un F1 global del 72% no es perfecto, pero puede parecer razonable. El problema aparece al mirar por regiones: el modelo funciona muy bien en Galicia y muy mal en Extremadura.

                                                    La pregunta ya no es solo si el modelo funciona técnicamente. La pregunta es si es justo desplegar un sistema que protege mejor a unos territorios que a otros.

                                                    Posición 1 Sí, desplegar

                                                    El modelo puede ser mejor que no tener nada. Aunque falle en algunas zonas, podría ayudar a prevenir parte del riesgo.

                                                    Posición 2 No, esperar

                                                    Un F1 del 38% en Extremadura puede ser inaceptable. Desplegarlo podría dejar peor protegida a una región concreta.

                                                    Posición 3 Depende

                                                    Depende del uso: si es apoyo para humanos, si se avisa de la baja fiabilidad regional y si existen medidas compensatorias.

                                                    Posición 4 Desplegar con advertencia

                                                    Se podría usar el modelo indicando explícitamente que en Extremadura su fiabilidad es baja y requiere revisión humana.

                                                    Guía para moderar

                                                    Da unos segundos para pensar. Pide una votación rápida: sí, no o depende. Después pide uno o dos argumentos. No hace falta cerrar con una respuesta única. El objetivo es que el grupo entienda que una métrica global puede ocultar daños locales.

                                                    Cierre

                                                    ¿Qué error estás dispuesto a aceptar?

                                                    Hemos jugado a ser un modelo de clasificación. Hemos visto que acertar mucho no basta. Accuracy puede mentir, Recall te dice si dejas escapar lo importante, Precision te dice si puedes confiar cuando dices “sí”, y F1 te obliga a equilibrar ambas.

                                                    Lo más importante no son las fórmulas

                                                    Lo importante es la pregunta que hay detrás: ¿qué error estás dispuesto a aceptar y quién paga el precio de ese error?

                                                    Cuando evaluéis un modelo en vuestros proyectos, no miréis solo el número. Mirad quién se beneficia de ese número y quién puede quedar perjudicado cuando el modelo se equivoca.

                                                    Accuracy Aciertos globales

                                                    Útil cuando las clases están equilibradas y todos los errores cuestan parecido.

                                                    Recall No dejar escapar lo importante

                                                    Clave cuando un falso negativo puede ser grave.

                                                    Precision No acusar sin fiabilidad

                                                    Clave cuando una falsa alarma puede dañar a una persona o generar costes altos.

                                                    F1-Score Equilibrio

                                                    Resume si el modelo detecta bien sin disparar demasiadas falsas alarmas.
